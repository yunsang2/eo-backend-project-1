### 2. 메인 화면 인터페이스 회의

#### 2-1. 전체적인 화면 구상

- 메인 화면과 검색 후 화면을 어떻게 표현할 것인가

  - 메인 화면에 변형을 주는 방식  
    - ❌ 채택되지 않음

  - 검색 시 새로운 페이지로 이동하는 방식  
    - ⭕ 채택됨

  - 질문 입력 전에는 화면 중앙에 질문 영역 배치  
  - 질문 내역 출력 시, 질문 영역을 제외한 콘텐츠가 중앙 정렬되도록 구성

---

#### 2-2. 검색 과정 및 답변 출력 방식

- 검색 텍스트와 답변을 화면에 표현하는 방식 논의

  - 검색 텍스트가 하단에서 상단으로 올라가는  
    카카오톡 채팅 형태  
    - ❌ 채택되지 않음

  - 검색 텍스트는 상단에 위치하고  
    답변은 하단에 출력되는 방식  
    - ⭕ 채택됨

  - 화면 크기가 줄어들 경우
    - 왼쪽 검색 내역 바를 닫고 중앙 정렬
    - 버튼 활성화 방식
    - 메인 화면과 검색 후 화면을 분리하면 구조가 복잡해질 수 있다는 의견
    - ❌ 채택되지 않음

---

#### 2-3. 도전 과제

- 검색 과정과 답변을 어떻게 표현할 것인가

  - 예상 질문지를 미리 출력하고  
    질문 선택 시 해당 답변을 출력하는 방식 검토

  - 발표 시 정해진 질문만 답변 출력
    - 정해지지 않은 질문 입력 시 오류 메시지 출력  
      - 예: “무슨 말씀하시는지 모르겠습니다. 다시 입력해 주세요.”
    - ❌ 채택되지 않음

## 작업 회고

- 윤상이: 팀 프로젝트를 처음 진행하면서 팀장직을 맡아 다른 분들이랑 작업하는게 신선하면서도 긴장되서 살짝 작업이 느려진점에대해 아쉬웠고 이후 팀원들이 담당한 내용만 신경써서 자신이 담당한 내용을 만들지 못해서 아쉽다
- 현길용: 팀원들과 협동과정이 재밌었고 아직 안배운 GitHub를 사용해 프로젝트를 진행하는것에 관해 어려움을 느꼈고 프로젝트로 인해 오랜만에 작성하는 HTML / CSS 을 다시 공부 할 필요를 느낌.
- 최서윤: 깃헙 사용에 지식이 부족해서 처음부터 협업과정이 어렵고 서투르게 느껴졌다. 피그마를 활용하여 전반적인 디자인을 먼저 만들고 큰 그림을 따라서 코드를 작성하니 훨씬 수월한 느낌이었고 배웠던 내용을 실습을 통해 복습할 수 있어서 좋았다.
