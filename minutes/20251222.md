## 1. 회의한 내용

- **윤상이**
  - 메인 페이지 JavaScript 수정 진행
  - 금일 헤더 및 푸터 구성에 대한 회의 진행

### 푸터 / 헤더 및 레이아웃 관련 논의

- **현길용**
  - 스크롤바를 생성하여 푸터를 기본적으로 숨기고,
    스크롤 시 보이도록 하자는 의견 → X

- **최서윤**
  - 푸터를 중앙 정렬로 배치하는 것도 하나의 방법
  - 일단 대기 상태로 제작 후 최종 결정하기로 함

- **윤상이**
  - 필요한 페이지에만 푸터를 선택적으로 적용
  - 푸터를 제외한 페이지들은 요소 크기가 작거나 차지하는 영역이 적어
    우측 하단에 붙이는 방식도 괜찮을 것 같다는 의견 → O

- 푸터에 포함될 요소 구성
  - **팀원 이름을 포함하기로 결정** → O

- 검색 결과 페이지 관련
  - 검색 결과 페이지 요소를 추가하여 더 다채로운 구성 시도 → O
  - **현길용**
    - 검색 결과 페이지의 푸터는
      컨텐츠 영역 기준 좌측 하단에 배치 → O
  - **최서윤**
    - 검색창을 헤더 영역에 포함시키는 것이 더 깔끔할 것 같다는 의견 → O

- 질문 결과 페이지 구성
  - **윤상이**
    - 사이드바를 추가하기보다는
      요소 전체 크기를 줄여 구성하자는 의견 → O

### 스타일 적용 방식 논의

- **윤상이**
  - 모든 페이지에 동일한 스타일을 적용하기보다
    - 검색 결과 페이지 전용 스타일
    - 그 외 페이지는 공통 CSS
  - 페이지 성격에 따라 스타일을 분리 적용하자는 의견 → O

- 푸터 표현 방식
  - **현길용**
    - 푸터를 구분선으로 나누는 방식 제안 → O
  - **윤상이**
    - 푸터를 박스 형태로 표현하는 방식 → X

## 6. 회고록

- **윤상이**
  - 지난주부터 문제였던 로그인 로직을
    쿠키를 사용해 해결하는 방법을 알게 되었으나,
    쿠키 개념이 익숙하지 않아 작업을 진행하지 못함
  - 쿠키에 대해 추가 학습 후
    다음 날 다시 구현을 시도하기로 함

- **현길용**
  - 헤더와 푸터의 공통화,
    그리고 전체 레이아웃 구조 정리에 집중한 하루였음
  - 공통 CSS는 작업 전에
    팀원들과 함께 먼저 설계했어야 했다고 느꼈음

- **최서윤**
  - 헤더와 푸터를 제작해 각 페이지에 적용하는 과정에서
    기존 페이지 레이아웃이 깨지는 문제가 발생함
  - 먼저 공통 레이아웃(헤더·푸터)을 설계한 뒤
    각 페이지를 작성하는 방식이 더 효율적이었을 것이라 판단
  - 더 나은 작업 순서와 구조에 대해 고민하는 계기가 되었음
